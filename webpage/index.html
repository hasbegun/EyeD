Core Capabilities

What EyeD Does

A complete iris recognition pipeline from capture to match, built for production deployments where privacy is non-negotiable.

👁

Real-Time Iris Analysis

Captures eye images via edge devices, applies quality gating, and runs the full segmentation-normalization-encoding-matching pipeline in under 88ms.

🔒

Encrypted Matching

Iris templates are encrypted using Fully Homomorphic Encryption (OpenFHE). Matching is performed on ciphertext — the server never sees your biometric data.

🔗

Secure Multi-Party Computation

3-party replicated secret sharing splits templates across independent parties. No single party can reconstruct the original iris code.

📦

Containerized Microservices

Six independent Docker services scale from a single Jetson Nano to a multi-GPU Kubernetes cluster. Deploy on-prem, in the cloud, or hybrid.

🌐

Edge-to-Cloud Architecture

Headless Raspberry Pi capture devices stream quality-gated frames to compute servers via gRPC. WebRTC provides live video monitoring in the browser.

📊

Live Dashboard

Flutter-based web UI shows real-time match results, enrollment management, gallery visualization, and device monitoring via WebSocket.

Algorithm Performance

150x More Accurate Than Legacy

EyeD uses Open-IRIS (Worldcoin, MIT) with DNN segmentation, replacing the legacy Masek pipeline entirely.

Aspect Legacy (Masek) EyeD (Open-IRIS)
Segmentation Hough circles (IoU ~35%) MobileNetV2 + UNet++ (IoU 94.3%)
Encoding 1D Log-Gabor, ~2K bits 2D Gabor, multi-scale, >10K bits
Matching Hamming distance (no mask) Masked fractional Hamming
FNMR @ FMR=0.001 ~17-27% 0.12% (150x better)
Eyelid Detection 1-2 minutes (curve fitting) Included in DNN (0ms extra)
Full Pipeline ~2+ minutes ~43ms (GPU) / ~130ms (CPU)
Privacy by Design

Cryptographic Security

Biometric data is the most sensitive data you have. EyeD treats it accordingly.

Fully Homomorphic Encryption

Iris templates are encrypted with OpenFHE (BFV/BGV schemes) before storage. Template matching runs entirely on ciphertext. The compute server never sees plaintext biometric data.

3-Party SMPC

Templates are split into secret shares distributed across three independent parties via replicated secret sharing. Reconstruction requires collusion of all three parties, which is architecturally prevented.

Secure Memory

Cryptographic keys use mlock-protected memory buffers that are zeroed on deallocation. No key material is ever written to swap or paged to disk.

Operational Modes

Three modes — prod (FHE locked ON, minimal API surface), dev (toggleable, full debug), and test (isolated DB). Prod is the safe default when no config is present.

Flexible Deployment

Runs Anywhere

From a single Jetson Nano to a multi-GPU Kubernetes cluster.

On-Prem (Jetson)

All 6 containers on one Jetson Xavier. Lowest latency, zero internet dependency. Air-gapped deployments supported.

Hybrid Edge + Cloud

Gateway + iris-engine in the cloud (AWS g4dn, GCP T4). Template DB stays on-prem for data sovereignty.

Full Cloud

All containers on Kubernetes. RPi capture devices connect over VPN. Auto-scale iris-engine replicas with load.

Development

Single docker compose up on Mac (Apple Silicon) or Linux. CPU-only mode, ~130ms inference. Fully functional.
